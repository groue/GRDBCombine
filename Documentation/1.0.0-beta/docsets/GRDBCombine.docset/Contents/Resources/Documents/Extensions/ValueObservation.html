ValueObservation

extension ValueObservation

Combine extensions on ValueObservation.

  • publisher(in:)

    Creates a publisher which tracks changes in database values.

    For example:

    let observation = ValueObservation.tracking { db in
    try Player.fetchAll(db)
}
let cancellable = observation
    .publisher(in: dbQueue)
    .sink(
        receiveCompletion: { completion in ... },
        receiveValue: { players: [Player] in
            print("fresh players: \(players)")
        })

    By default, fresh values are dispatched asynchronously on the main queue. You can change this behavior by calling the scheduling(_:) method on the returned publisher.

    For example, scheduling(.immediate) notifies all values on the main queue as well, and the first one is immediately notified when the publisher is subscribed:

    let cancellable = observation
    .publisher(in: dbQueue)
    .scheduling(.immediate) // <-
    .sink(
        receiveCompletion: { completion in ... },
        receiveValue: { players: [Player] in
            print("fresh players: \(players)")
        })
// <- here "fresh players" is already printed.
